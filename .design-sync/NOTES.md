# design-sync notes — Cura

Repo-specific gotchas for syncing Cura's design system to claude.ai/design.
Append a bullet whenever a sync teaches something the next run should know.

## Shape & entry
- **Shape is `package`, synth-entry mode.** Cura is an *app* (`package.json` is
  `"private": true` with no `module`/`main`/`exports`), not a published component
  library. There is no library `dist/` entry — `dist/` is the built PWA. The
  converter therefore synthesizes an entry from `src/` (`[NO_DIST]` on stderr is
  expected, not an error).
- The build MUST be invoked with a `--entry` that anchors `PKG_DIR` at the repo
  root but doesn't exist as a real dist entry, e.g. `--entry ./dist/index.js`.
  Without any `--entry`, `PKG_DIR` resolves to `node_modules/cura` (which doesn't
  exist) and `srcDir` can't be found. The nonexistent path triggers synth mode
  and walks up to the `cura` package.json.
- `srcDir` is scoped to `src/app/components` — the actual design system per
  CLAUDE.md §7 (`shared.tsx` primitives + custom components + `ui/` shadcn
  primitives). Do NOT widen to `src/` or `src/app`: that pulls feature pages,
  stores, and the data layer in as "components".
- Components are cleanly presentational — none import `useCuraStore`, the data
  layer, or Supabase — so the bundle stays lean and `import.meta.env.VITE_*`
  resolves to `undefined` under the IIFE define (store defaults to local mode,
  no network).

## Styling (Tailwind v4)
- Tokens live in `src/styles/theme.css` (`:root` custom props + `@theme inline`),
  utilities are generated by Tailwind v4 (`@tailwindcss/vite`). The RAW source
  CSS is uncompiled (no generated utility classes), so `cssEntry` must point at a
  COMPILED stylesheet.
- `cssEntry` = `.design-sync/ds-compiled.css`, a committed copy of the app's own
  compiled Tailwind bundle (`dist/assets/index-<hash>.css`). It contains every
  `:root` token, the `@theme`-generated utilities, `@utility focus-ring`, the
  keyframes, and `tw-animate-css` — a superset of what the DS components use
  (compiled from the whole app).
- **Re-sync risk (CSS staleness):** `.design-sync/ds-compiled.css` is a snapshot.
  If component styling/source changed since it was captured, refresh it before
  building: `pnpm build && cp dist/assets/index-*.css .design-sync/ds-compiled.css`
  (the app CSS filename is content-hashed, hence the glob + stable copy).
  `dist/` is gitignored; the stable copy is committed so a clean re-sync with no
  source change already has current CSS.
- NOTE: `pnpm-workspace.yaml` pins `supportedArchitectures: os: linux`, so this
  repo's `node_modules` may carry Linux native binaries. `pnpm build` on Windows
  can fail on native deps (oxide/lightningcss). The converter does NOT need
  `pnpm build` — it bundles from `src/` with its own isolated esbuild and only
  needs the pre-compiled CSS, which is already captured.

## Fonts
- Fonts (Lora, Plus Jakarta Sans) are loaded via a REMOTE Google Fonts
  `@import url(https://fonts.googleapis.com/...)` at the top of the compiled CSS.
  Expect `[FONT_REMOTE]` (informational). **Re-sync risk:** if the design
  environment blocks external font hosts, self-host the two families as
  `@font-face` + woff2 via `cfg.extraFonts` instead.

## Providers
- `RoutineKaart` and `FocusMiniPill` call `useNavigate` from `react-router` — they
  need a Router context or they render blank. Handled per-preview with an
  in-preview `<MemoryRouter>` wrapper (not a global `cfg.provider`, so other
  cards aren't needlessly wrapped).
- `ConnectivityBanner` / `UpdatePrompt` are sonner-toast triggers / app-shell
  chrome — they render little standalone UI; likely floor-card or minimal preview.

## Re-sync risks (watch-list)
- CSS snapshot staleness (see Styling above).
- Remote font host may be blocked in some environments (see Fonts above).
- Synth-entry `.d.ts` contracts are derived by ts-morph from `src/` (no shipped
  types) — generics/complex prop types may be weaker than a real build would
  give. Check `[DTS_*]` warnings on each run.

## CRITICAL capture-harness patches — RE-APPLY ON EVERY RE-SYNC
The staged `.ds-sync/` is re-copied from the skill each sync, so these two edits
must be re-applied or many components capture BLANK (false `[RENDER_BLANK]` +
`needs-work` grades). Cura components animate in with Framer Motion
`initial={{opacity:0}}` / `{scale:0}` mount tweens (~0.22–0.32s):
1. **`.ds-sync/package-capture.mjs`** — comment out the
   `page.clock.setFixedTime(...)` line. Playwright's fixed clock freezes the time
   source Framer's duration tweens read, so opacity-0 mount fades never progress
   → cards capture at opacity 0 (blank). Cura previews pass dates as literals, so
   no determinism is lost. (Also added `page.waitForTimeout(600)` inside `settle()`.)
2. **`.ds-sync/package-validate.mjs`** — add `await page.waitForTimeout(650)`
   right after the render-check `page.goto(... 'networkidle' ...)` (before the
   `rootEmpty` evaluate), so the mount fade settles before the screenshot.
Symptom if missing: PageHeader, Leeg, and every list-row/feed component
(SuggestieRij, BoodschapRij, ActiviteitReacties, TijdlijnTaakRij) render blank.

## public/ assets via absolute paths — NOT renderable in the sandbox (by design)
The capture server and the DS upload serve only `ds-bundle/`, which has no
`public/` assets, and the converter does not ship arbitrary public assets.
Components that load images via absolute `/…` paths therefore can't show art:
- **Room art** (`public/rooms/*.png`): KamerKaart / KamerKunstKiezer / RoomArt /
  RoomHero / RoomThumb render their DESIGNED FALLBACK (tinted wash + line icon) —
  a legitimate art-less-room state (CLAUDE.md §3 "degrade gracefully"). Authored
  and graded on the fallback.
- **Image-only decorative components** — AppBackground, PageBanner, LandingHeader,
  EmptyIllustration, and **Logo** (`<img src="/logo.svg">`, only size/className
  props) render empty/null without the asset → left as documented FLOOR CARDS
  (still fully importable + `.d.ts`/`.prompt.md`).
- In a real Cura codebase these assets ARE served from `public/` at `/`, so the
  design agent's generated code renders them correctly — only the design-env
  PREVIEW can't. To bring them off the floor on a future sync: FIRST confirm
  claude.ai/design serves project-root assets at `/`, then copy `Cura/public/*`
  into `ds-bundle/` post-build and add those paths to the upload plan.

## Preview-authoring patterns that worked (for future authoring)
- Tiny primitives read best as a labelled row/group in ONE cell (variant axis
  side by side); decorative (aria-hidden) components (Avatar, RingProgress) need a
  visible companion label (name / %).
- `.design-sync/ds-compiled.css` is comprehensive — every utility the components
  use (incl. arbitrary values like `text-[0.9375rem]`) is available in previews.
- Field inputs wrap well at `width: 340`; full-width buttons at `width: 320`.
- GroupCard takes a literal ARRAY of InstRij children (needs `key`). OptieKaart's
  selected state needs bumped `selectedBg`/`selectedBorder` to read clearly.
- Popover: author only the composed `Popover` (`defaultOpen` + Trigger + Content
  inside an overlay Frame); PopoverAnchor/Content/Trigger stay floor cards.
- Overlay/full-screen components (Sheet, Popover, FullScreenError, FullScreenSkeleton)
  use `cardMode: "single"` + a declared viewport; row/card components use
  `cardMode: "column"`.

## Wave-2 findings (durable)
- **`react-router` MUST be in `cfg.extraEntries`.** RoutineKaart / RoutineKaartCompact
  / FocusMiniPill call `useNavigate`. The bundler externalizes only react/react-dom
  to `window.*`; react-router is bundled fresh, so an in-preview `<MemoryRouter>`
  gets a SECOND react-router instance → its `NavigationContext` differs from the one
  the components (in `window.Cura`) read → `useNavigate` throws → blank card. Fix
  (applied): `"extraEntries": ["react-router"]` merges react-router onto `window.Cura`
  using the bundle's single instance, so a preview's `import { MemoryRouter } from
  'react-router'` resolves to the SAME instance. (General rule: any lib whose React
  context a component reads must be shared this way.)
- **The compiled CSS is tree-shaken to Cura's own used utilities** — `ds-compiled.css`
  was built by scanning `src/**`, so a Tailwind class that never appears in the app
  (e.g. `w-28`, `h-28`) is ABSENT and silently no-ops in previews AND in any design
  built with the DS. Author previews with source-present classes or inline pixel
  `style`. Confirmed present: `w-16/w-20/w-full/h-full`, `rounded-2xl/3xl/full`,
  `px-5`, `pt-14`, `bg-card`, `bg-card-active`, `border-border`, `text-*`, `space-y-*`.
- **BoodschapRij** signature is `{ item, onToggle, onDelete, onUpdate }` — `onUpdate`
  is REQUIRED (steppers/inline edit), not optional.
- **RoomHero** without art renders only its floating back/edit control bar (its
  watercolor header can't load) — sparse but honest; candidate for a future art
  treatment if public assets ever ship.
- **Do a full `package-build` after ANY `cfg.overrides` change before a scoped
  `preview-rebuild`** — a changed override re-keys that component's stamped grade
  slice, and preview-rebuild aborts `[CONFIG_STALE]` until the stamp is refreshed.
- **`cfg.extraEntries` is in the GLOBAL grade-key slice** — adding/removing it
  re-keys EVERY component's grade (all grades clear on the next capture). Set it
  ONCE, early. (Adding `react-router` mid-run forced a full re-grade this sync.)
- **Full builds are slow (~5–7 min)** because `react-router` (an extraEntry) has a
  large `.d.ts` tree that ts-morph parses on every build. Background the build.

## Known render warns (legitimate — re-sync must not read these as new)
- **`[FONT_REMOTE]` Plus Jakarta Sans, Lora** — fonts load via a remote Google
  Fonts `@import`; expected (see Fonts section).
- **`[RENDER_THIN]` FullScreenError, FullScreenSkeleton** — their `fixed inset-0`
  layout collapses the grid-view height measurement to 0px, so validate flags them
  thin. Their SOLO `?story=` captures render full-bleed correctly (they're
  `cardMode: single`); the thin flag is a grid-measurement artifact, not a real
  blank. Benign.
- **Logo renders a broken-image glyph** — it's `<img src="/logo.svg">` and the
  asset 404s in the sandbox (see the public/ assets section). It's importable +
  documented; in a real Cura app the logo renders. Not floor-carded because its
  `<img>` element makes the root non-empty (so the typographic floor doesn't kick
  in). Left as-is.
